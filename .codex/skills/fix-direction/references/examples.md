# Bug Fix Examples

## 起動例
- `翻訳開始ボタンが反応しないのを修正して`
- `バグ: 保存後に一覧が更新されない`
- `バグ: 再開後に進捗表示が消えたままになる`
- `この不具合、docs と実装がズレていないか見て`

## 起動しない例
- `新しい翻訳設定画面を設計して`
- `Master Persona のシナリオを整理して`
- `docs/ のタイトルを見直して`

## UI バグの例
依頼:
`バグ: 実行中に Cancel を押してもボタンが disabled のままで戻らない`

期待される進め方:
- 対象画面を特定する
- `docs/` の関連 spec を確認する
- `wails dev` と Playwright MCP で再現する
- 必要なら最小限のログを追加して state 遷移を観測する
- 修正後に同じ手順で回帰確認する

## Backend バグの例
依頼:
`辞書作成が途中で止まるのを修正して`

期待される進め方:
- 再現条件と停止地点を固定する
- 関連 spec とログを照合する
- 実装不備か、workflow 想定との乖離かを切り分ける
- 対象ファイル単位の lint から直し、最後に backend 品質ゲートを通す

## 仕様乖離の例
依頼:
`バグ: 翻訳完了後に結果パネルが自動で開かない`

切り分け観点:
- `docs/` に自動展開が明記されているなら実装不備
- 現行要件では自動展開しないなら、報告内容ではなく spec を優先する
- spec が曖昧ならユーザー確認を挟む
