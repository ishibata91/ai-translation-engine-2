# Impl Direction Templates

## 下流スキル起動
```md
### Skill Invocation
- invoked_skill: <起動する下流スキル名 (例: impl-distill)>
- invoked_by: impl-direction
- agent: <起動するエージェント名 (例: ctx_loader)>
- 対象 change:
- task:
- 入力:
- focus:
```

> 下流スキルのサブエージェントを立ち上げるときは必ずこのテンプレートを使い、
> `invoked_skill` と `invoked_by` を明示すること。
> サブエージェントは自分がどのスキルで起動されたかをこのフィールドで確認する。

## section plan 受領
`impl-workplan` の packet 返却後、worker を起動する前に必ずこのテンプレートで記録する。

```md
### Workplan Summary
- change:
- tasks_path:
- shared_contracts:
- dispatch_order:
- sections:
  - section_id:
    title:
    owner: frontend | backend
    goal:
    depends_on:
    shared_contract:
    required_reading:
    owned_paths:
    forbidden_paths:
    validation_commands:
    acceptance:
```

> `Workplan Summary` の `sections` は `impl-workplan` の `Section Plan` schema をそのまま写す。
> `title` `shared_contract` `required_reading` `validation_commands` `acceptance` を省略しないこと。

## section dispatch
```md
### Section Dispatch
- change:
- section_id:
- title:
- owner: frontend | backend
- goal:
- depends_on:
- invoked_skill:
- agent: implementer
- shared_contract:
- required_reading:
- owned_paths:
- forbidden_paths:
- validation_commands:
- acceptance:
```

> `Section Dispatch` は `impl-workplan` の `Work Order` schema を包む dispatch envelope。
> `title` `goal` `depends_on` `shared_contract` `owned_paths` `forbidden_paths` `required_reading`
> `validation_commands` `acceptance` を workplan 返却から書き換えずに引き継ぐこと。

## review 差し戻し
```md
### Review Reroute
- change:
- review_score:
- affected_sections:
  - section_id:
    title:
    owner: frontend | backend
    goal:
    depends_on:
    shared_contract:
    required_reading:
    owned_paths:
    forbidden_paths:
    validation_commands:
    acceptance:
- required_delta:
- carry_over_contract:
- recheck:
```

> `affected_sections` に含まれる section だけを再 dispatch し、元の full work order を崩さない。
> `carry_over_contract` には `title` `goal` `depends_on` `shared_contract` `required_reading` `owned_paths` `forbidden_paths` `validation_commands` `acceptance` の再利用方針を書く。
