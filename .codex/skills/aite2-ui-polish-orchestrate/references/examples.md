# Design Fix Examples

この reference は `impl-direction` が既存 UI polish を specialized branch として切り出すときの internal helper 例を示す。独立した user-facing 入口の起動例ではない。

## impl-direction から specialized branch として渡す例
- `この画面の余白を直して`
- `一覧のラベルが切れているのを直して`
- `ボタン位置がずれて見えるので直して`
- `このダイアログの配置だけ整えて`

## 独立入口としては起動しない例
- `この画面の仕様を設計して`
- `この機能のシナリオを整理して`
- `保存処理が動かないのを直して`

## レイアウト崩れの例
依頼:
`一覧カードの高さがバラついて見えるのを直して`

期待される進め方:
- 画面を開いて見た目の崩れを観測する
- 対象要素と影響範囲を絞る
- レイアウト修正だけで直す
- 修正後に同じ画面で再確認する

## 視認性の例
依頼:
`補助テキストのコントラストが弱くて読みにくい`

期待される進め方:
- 問題箇所をスクリーンショットと snapshot で観測する
- 色、サイズ、余白、整列のどれが主因かを切り分ける
- ロジックを触らず見た目だけ修正する
