# ダッシュボード

## Purpose
TBD: アプリケーションの全体像を把握するためのダッシュボード画面に関する振る舞いを定義します。

## Requirements

### Requirement: Active Jobs Dashboard Overview
ダッシュボード画面では、起動時またはデフォルトの画面として、全体の進行中ジョブ・ステータスをテーブル形式で一覧表示しなければならない。

#### Scenario: Initial Dashboard Rendering
- **WHEN** メインコンテンツ領域でダッシュボードを表示するルート (`/`) にアクセスする
- **THEN** 「進行中のジョブ」を含むステータスカード領域が表示される
- **AND** ジョブの一覧として、対象（Mod名）、フェーズ（進捗状態バッジ）、詳細な進捗バー（%表示）が表示される

### Requirement: ダッシュボードは task 削除アクションを提供しなければならない
ダッシュボードは、停止済み・失敗済み・再開待ちの task に対して削除アクションを表示し、ユーザーが不要な task を一覧から除去できるようにしなければならない。削除アクションは実行中 task では利用可能にしてはならない。

#### Scenario: 削除可能な task に削除ボタンを表示する
- **WHEN** ダッシュボードが `pending`、`paused`、`request_generated`、`failed`、`cancelled` のいずれかの status を持つ task を表示する
- **THEN** システムは当該行のアクション列に task 削除ボタンを表示しなければならない
- **AND** `running` status の task には task 削除ボタンを表示してはならない

#### Scenario: 削除前に確認する
- **WHEN** ユーザーが task 削除ボタンを押す
- **THEN** システムは task 本体と task ごとの中間成果物が削除されることを明示した確認導線を表示しなければならない
- **AND** ユーザーが確認するまで task を削除してはならない

#### Scenario: 削除完了後に一覧を更新する
- **WHEN** ユーザーが task 削除を確定し、削除処理が成功する
- **THEN** システムは削除した task をダッシュボード一覧から取り除かなければならない
- **AND** 削除済み task を再描画や再取得で再表示してはならない
