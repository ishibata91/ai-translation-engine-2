# Review Examples

## 起動例
- `この差分をレビューして`
- `仕様と実装のズレを見て`
- `自己レビューして`
- `docs と change に対して取りこぼしがないか確認して`

## 起動しない例
- `この change を実装して`
- `UI 契約を決めたい`
- `この画面の余白を直して`

## finding の例
- バグ: 再試行時に state が戻らず回帰している
- 責務逸脱: controller が workflow の進行判断を持っている
- 仕様未反映: `ui.md` の state が実装に反映されていない
- テスト不足: 主要異常系の確認がない
