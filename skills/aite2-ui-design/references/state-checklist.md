# UI State Checklist

## 先に決めること
1. Purpose は何か
2. Primary Action は何か
3. 主要 state は何か
4. state ごとに何が見えて、何が操作不能になるか
5. 成功、失敗、空、待機、再試行をどう扱うか

## state ごとに書くこと
- 表示される主要要素
- 無効化される操作
- 遷移トリガー
- 成功時に消える要素
- 異常時に残る要素

## 確認時の注意
- state を文章だけで済ませない
- loading / error / empty を省略しない
- UI state と内部処理状態を無造作に混ぜない
- 画面構造を定義せずに場当たりで組み立てない
