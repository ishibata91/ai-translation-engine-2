# Design Routing Examples

## 既存見た目修正
- 依頼: `この画面の余白を直して`
- 最初の skill: `aite2-ui-polish`

## UI を含む新規変更
- 依頼: `新しい設定画面の仕様を決めたい`
- 最初の skill: `aite2-ui-design`
- 続き: `aite2-scenario-design` -> `aite2-logic-design`

## UI を含まない仕様変更
- 依頼: `再試行フローのシナリオと内部責務を整理したい`
- 最初の skill: `aite2-scenario-design`
- 続き: `aite2-logic-design`

## docs 同期
- 依頼: `change 文書を docs に反映したい`
- 最初の skill: `aite2-sync-docs`
