## ADDED Requirements

### Requirement: 翻訳フローはデータロードフェーズから開始しなければならない
システムは、`TranslationFlow` の先頭フェーズとしてデータロードを表示し、既存の用語、ペルソナ、要約、翻訳、エクスポートより前に配置しなければならない。

#### Scenario: 翻訳フロー初期表示
- **WHEN** ユーザーが翻訳フロー画面を開く
- **THEN** システムはステップ表示とタブ表示の両方で `データロード` を先頭に表示しなければならない
- **AND** システムは初期アクティブフェーズを `データロード` にしなければならない

#### Scenario: ロード完了後の次フェーズ遷移
- **WHEN** ユーザーがデータロードフェーズで有効なロード結果を確認して次へ進む
- **THEN** システムは次のフェーズとして `用語` をアクティブにしなければならない

### Requirement: データロードフェーズは複数ファイル選択を受け付けなければならない
システムは、抽出データのロード入力として複数ファイル選択を受け付け、選択済みファイル一覧をユーザーへ提示しなければならない。

#### Scenario: 複数ファイルを選択する
- **WHEN** ユーザーがデータロードフェーズでファイル選択を実行し、2 件以上の抽出データファイルを選ぶ
- **THEN** システムは選択した全ファイルをロード対象一覧へ追加しなければならない
- **AND** システムは各ファイルを個別の入力項目として識別できる状態で保持しなければならない

#### Scenario: 選択済みファイルを確認する
- **WHEN** ユーザーがファイル選択後にデータロードフェーズを閲覧する
- **THEN** システムは各ファイルの名前またはパスを一覧表示しなければならない
- **AND** システムは未ロード状態とロード済み状態を区別できる UI を提供しなければならない

### Requirement: データロードフェーズは artifact に保存された全 section の翻訳対象をファイル単位で表示しなければならない
システムは、ロード処理で artifact に保存されたパース済みデータを読み出し、全 section の翻訳対象行を `DataTable` でファイルごとに独立した一覧として表示しなければならない。システムは複数ファイルの行データを 1 つのテーブルへマージしてはならない。

#### Scenario: ロード済みファイルを一覧表示する
- **WHEN** ユーザーが 1 件以上の抽出データファイルをロードし、artifact にパース済みデータが保存されている
- **THEN** システムは保存済みファイルごとに 1 つの `DataTable` を表示しなければならない
- **AND** 各 `DataTable` は対応するファイル名と合計件数を識別できる情報を表示しなければならない

#### Scenario: 全 section の preview 行を含める
- **WHEN** システムがファイルごとの preview 行を構築する
- **THEN** システムは dialogue response, quest stage, quest objective, item, magic, location, cell, system, message, load screen, NPC など翻訳対象となる全 section を preview 対象に含めなければならない
- **AND** システムは特定 section だけを preview 対象として固定してはならない

#### Scenario: 複数ファイルをロードしたときにテーブルを分離する
- **WHEN** ユーザーが複数ファイルをロードする
- **THEN** システムはファイル A の行とファイル B の行を別々の `DataTable` に表示しなければならない
- **AND** システムはファイル境界を失う形で全行を単一テーブルへ統合してはならない

### Requirement: ファイルごとの翻訳対象テーブルは 50 行単位でページングできなければならない
システムは、データロードフェーズで表示する各ファイルの `DataTable` について、1 ページあたり 50 行の preview を表示し、ページ切り替えを可能にしなければならない。

#### Scenario: preview をページ切り替えする
- **WHEN** ユーザーがファイルテーブルのページを切り替える
- **THEN** システムは対象ファイルの preview 行を 50 行単位で切り替えて表示しなければならない
- **AND** 他のファイルテーブルの現在ページを勝手に変更してはならない

### Requirement: ファイルごとの翻訳対象テーブルは折りたたみ可能でなければならない
システムは、データロードフェーズで表示する各ファイルの `DataTable` を個別に折りたたみ可能にしなければならない。

#### Scenario: 個別テーブルを折りたたむ
- **WHEN** ユーザーが特定ファイルのテーブル見出しを操作する
- **THEN** システムはそのファイルの `DataTable` だけを折りたたみまたは展開しなければならない
- **AND** 他のファイルの `DataTable` の展開状態を勝手に変更してはならない
