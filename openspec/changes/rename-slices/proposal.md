# Proposal: 各スライスのリネームと仕様書のメンテナンス

本プロポーザルでは、システムの認知負荷を下げ、ドメイン用語に合わせるために、各スライス（モジュール）の名前を刷新し、それに伴う仕様書およびコードベースのメンテナンスを行う。

## 背景・目的
現在のスライス名（`LoaderSlice`, `Pass2TranslatorSlice` 等）は、開発初期の仮称や実装寄りの名前が多く、システム全体の役割を直感的に表せていない。
より短く、かつ役割が明確な名前に統合・変更することで、コードの可読性とメンテナンス性を向上させる。

## 変更内容
以下の通り、スライス名およびパッケージ構成を変更する。

### スライス名のマッピング
| 現行名 (Spec/Package) | 変更後 (New Name) | 備考                             |
| :-------------------- | :---------------- | :------------------------------- |
| `config-store`        | `config`          | 設定管理                         |
| `loader`              | `parser`          | XML/JSON解析と抽出               |
| `context-engine`      | `lore`            | 過去の翻訳文脈やRAGの検索        |
| `persona-gen`         | `persona`         | NPCの口調・人格解析              |
| `dictionary-builder`  | `dictionary`      | 固有名詞辞書の構築               |
| `term-translator`     | `terminology`     | 固有名詞辞書の適用               |
| `pass2-translator`    | `translator`      | LLMによるメインの翻訳処理        |
| `summary-generator`   | `summary`         | 文脈の圧縮・要約                 |
| `process-manager`     | `pipeline`        | 全体ジョブのオーケストレーション |

※ `llm-client` などのインフラ層については、現状維持とする。

### 影響範囲
- **仕様書 (`specs/`)**: 各スライスのディレクトリ名および `spec.md` 内の記述を更新。
- **ソースコード (`pkg/`)**: パッケージ名およびディレクトリ名を変更。
- **依存関係**: 各パッケージ間の `import` パス、`wire` によるDI定義等を一括置換。
- **ドキュメント**: `requirements.md`, `database_erd.md`, `refactoring_strategy.md` 内の参照を更新。

## 解放される機能 (Unlocks)
- パッケージ構成の簡素化による開発効率の向上
- ドメイン用語の統一によるコミュニケーションコストの削減

## 成功基準
1. `specs/` 下のディレクトリ構成が新しい定義と一致していること。
2. 全ての `spec.md` 内で古い名称が残っていないこと。
3. `requirements.md` 等の共通ドキュメントが更新されていること。
4. (実装フェーズにおいて) 全ての `pkg/` 構成が変更され、ビルドが通ること。
