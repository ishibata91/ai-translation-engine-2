# Proposal: Dictionary UI Improvements

## Motivation
現在、辞書エントリ一覧画面において、データ件数（例えば10,000件など）が膨大になるとフロントエンドのレンダリングパフォーマンスが著しく低下し、動作が重くなる問題が発生している。このパフォーマンス問題を解消し、快適なユーザー体験を提供する必要がある。
また、XMLインポートのUI領域が、未インポート（初期状態）のときでも縦に長すぎるため、画面の表示領域を不要に占有している。これをコンパクトな状態に縮小し、インポート中のみ広がるように改善する。
さらに、辞書の利便性を高めるため、単一の辞書ソースだけでなく、登録されているすべての辞書ソースを対象とした横断検索機能を新たに追加する。

## Capabilities
- `cross-source-search`: 全ての辞書ソースを対象とした横断検索機能を提供する。どの辞書に特定の用語が登録されているかを一括で検索・確認できる仕組み。（表示の仕方やUIなどの詳細仕様は別途検討）

## Modified Capabilities
- `dictionary-entry-list`: 辞書エントリの一覧表示にて、500件ごとのページング処理（または仮想スクロールによる段階的読み込み）を導入する。なお、検索（フィルタリング）自体は現在表示されているページ内だけでなく、バックエンド側で全データを対象に行う仕様に変更する。
- `dictionary-import-ui`: XMLインポートのUIレイアウトを変更する。データを選択・待機している初期状態では視覚的にコンパクトにまとめ、インポート処理中や結果表示時のみ縦ยาวに展開されるようにする。

## Impact
- **Frontend**
  - `frontend/src/pages/DictionaryBuilder.tsx`: エントリ一覧のページネーション対応、インポートUIのコンポーネントにおける高さ調整（Tailwind/CSS設定変更）、横断検索UIの追加。
- **Backend (Go)**
  - `pkg/dictionary/contract.go` (Wails API): エントリ取得メソッドにページネーションパラメータ（limit, offset）の追加や、全データ対象の検索パラメータの追加。全ソース横断検索APIの追加。
  - `pkg/dictionary/` 配下のServiceおよびRepository層: データベースからエントリを取得する際のLIMIT/OFFSETを利用したページネーション対応。全辞書対象の検索クエリ実装。
