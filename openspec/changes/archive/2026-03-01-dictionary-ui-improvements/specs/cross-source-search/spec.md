## ADDED Requirements

### Requirement: 全ソース（横断）検索機能の提供
ユーザーは、すべての登録済み辞書ソースを横断して、特定のキーワード（単語、訳語など）を検索するための機能を利用できなければならない（SHALL）。
横断検索は専用の「検索モーダル」を介して実行され、モーダル内で「検索条件」および「検索対象となる特定のソースの選択（または全ソースの選択）」が可能でなければならない（SHALL）。

#### Scenario: 検索モーダルによる条件設定とソース選択
- **WHEN** ユーザーが辞書UI等から横断検索へのトリガーボタン（検索アイコン等）をクリックしたとき
- **THEN** 検索項目（テキスト等）および対象のソース一覧を選択できる「検索モーダル」が表示されること
- **WHEN** モーダル上で条件を設定し「検索実行」を押下したとき
- **THEN** モーダルが閉じ、検索結果の一覧ビューへ遷移・推移すること

### Requirement: 検索結果の一時的表示（GridEditorと戻る・クリアボタン連携）
横断検索の実行結果は、専用の一時的なUI（`mocks-react/src/components/GridEditor.tsx`、もしくは同等のGridEditorコンポーネントをベースとした表示）として描画されなければならない（SHALL）。
この表示領域においては、以下の動作を含んでいなければならない（SHALL）。
1. 単一ソースの辞書表示と同様の **500件単位でのページネーション（サーバーサイド制御）** による表示を行うこと。
2. 検索結果として表示されたエントリ群に対しては、**一覧ビューから直接インライン編集および削除操作が可能でなければならない（SHALL）**。
3. 行削除が行われた場合を含め、あらゆる編集が行われた時点で「未保存（編集中）状態」と見なさなければならない（SHALL）。未保存の変更がある状態で通常の画面へ復帰しようと「戻る」ボタンを押下した際には、変更が破棄される旨の警告を表示し、ユーザーによる確認を要求しなければならない（SHALL）。また、未保存操作を取り消す「もとに戻す」機能を提供しなければならない（SHALL）。
4. その検索結果に対してGridEditor上で「さらに絞り込む」ためのカラムフィルタを利用できること。このフィルタは文字入力時ではなく **「検索」ボタン押下によってのみ反映・再取得** され、「クリア」ボタン押下によってフィルタ無しの元の横断検索結果状態へ戻ること。（＝単一ソース一覧と共通のフィルタ動作）
5. 画面上に「戻る」ボタンが配置され、押下時（および必要に応じた警告確認後）には横断検索結果の表示を破棄して、通常の元の画面（親の辞書ソース一覧など）へ復帰・遷移できなければならない（SHALL）。

#### Scenario: 一時的なGridEditorでの結果表示と戻る操作
- **WHEN** 検索モーダルから検索を実行したとき
- **THEN** GridEditorを利用した一時的な検索結果ビューに遷移し、一度に500件までのページネーション表示が行われること。このとき、各エントリが「どの辞書ソースに属しているか」が判別できる情報が含まれているとなお良い
- **WHEN** エントリが大量にある状態のとき
- **THEN** 次ページ、前ページへの遷移をサポートするページネーションUI操作が提供されること
- **WHEN** 結果表示画面で「戻る」ボタンをクリックしたとき
- **THEN** 検索結果ビューが完全に破棄され、検索を実行する前までの画面状態へ戻ること

#### Scenario: 検索結果に対するさらなる絞り込みとクリア
- **WHEN** 横断検索結果のGridEditor上で、カラムフィルタ条件を入力後「検索ボタン」をクリックしたとき
- **THEN** 今表示されている横断検索結果からさらに追加フィルタが適用されて（1文字ごとのリアルタイム反映ではなく）表示が更新されること
- **WHEN** さらに「クリアボタン」をクリックしたとき
- **THEN** 追加したフィルタのみが解除・無視され、元の横断検索結果（ページネーション状態）へ再度表示が戻ること
