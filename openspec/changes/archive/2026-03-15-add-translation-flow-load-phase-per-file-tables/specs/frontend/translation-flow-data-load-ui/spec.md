## ADDED Requirements

### Requirement: Translation Flow UI はデータロード画面を先頭フェーズとして表示しなければならない
システムは、`TranslationFlow` の UI において `データロード` を先頭フェーズとして表示し、ステップ表示とタブ表示の両方で最初に提示しなければならない。UI は workflow が管理する phase 順序を前提に描画し、先頭フェーズを別画面へ逃がしてはならない。

#### Scenario: 翻訳フロー初期表示でデータロード画面を表示する
- **WHEN** ユーザーが翻訳フロー画面を開く
- **THEN** UI は `データロード` を最初のフェーズとして表示しなければならない
- **AND** メインコンテンツにはデータロード用の入力と結果確認 UI が表示されなければならない

### Requirement: データロード画面は複数ファイル選択を受け付けなければならない
システムは、データロード画面で複数ファイル選択を受け付け、選択済みファイル一覧をユーザーへ提示しなければならない。

#### Scenario: 複数ファイルを選択する
- **WHEN** ユーザーがデータロード画面で 2 件以上の抽出データファイルを選ぶ
- **THEN** UI は選択した全ファイルをロード対象一覧へ追加しなければならない
- **AND** 各ファイルを個別の入力項目として識別できなければならない

#### Scenario: 選択済みファイルを確認する
- **WHEN** ユーザーがファイル選択後にデータロード画面を閲覧する
- **THEN** UI は各ファイルの名前またはパスを一覧表示しなければならない
- **AND** 未ロード状態とロード済み状態を区別できる表現を提供しなければならない

### Requirement: データロード画面は artifact preview をファイル単位のテーブルで表示しなければならない
システムは、artifact に保存されたパース済みデータの preview を、全 section を含むファイル単位の `DataTable` として表示しなければならない。システムは複数ファイルの行データを 1 つのテーブルへマージしてはならない。

#### Scenario: ロード済みファイルを一覧表示する
- **WHEN** ユーザーが 1 件以上の抽出データファイルをロードし、artifact にパース済みデータが保存されている
- **THEN** UI は保存済みファイルごとに 1 つの `DataTable` を表示しなければならない
- **AND** 各 `DataTable` は対応するファイル名と合計件数を識別できなければならない

#### Scenario: 全 section の preview 行を含める
- **WHEN** UI がファイルごとの preview 行を構築する
- **THEN** UI は dialogue response、quest stage、quest objective、item、magic、location、cell、system、message、load screen、NPC など翻訳対象となる全 section を preview 対象に含めなければならない
- **AND** 特定 section だけを preview 対象として固定してはならない

#### Scenario: 複数ファイルをロードしたときにテーブルを分離する
- **WHEN** ユーザーが複数ファイルをロードする
- **THEN** UI はファイル A の行とファイル B の行を別々の `DataTable` に表示しなければならない
- **AND** ファイル境界を失う形で全行を単一テーブルへ統合してはならない

### Requirement: ファイルごとの preview テーブルは 50 行単位でページングできなければならない
システムは、データロード画面で表示する各ファイルの `DataTable` について、1 ページあたり 50 行の preview を表示し、ページ切り替えを可能にしなければならない。

#### Scenario: preview をページ切り替えする
- **WHEN** ユーザーがファイルテーブルのページを切り替える
- **THEN** UI は対象ファイルの preview 行を 50 行単位で切り替えて表示しなければならない
- **AND** 他のファイルテーブルの現在ページを勝手に変更してはならない

### Requirement: ファイルごとの preview テーブルは折りたたみ可能でなければならない
システムは、データロード画面で表示する各ファイルの `DataTable` を個別に折りたたみ可能にしなければならない。

#### Scenario: 個別テーブルを折りたたむ
- **WHEN** ユーザーが特定ファイルのテーブル見出しを操作する
- **THEN** UI はそのファイルの `DataTable` だけを折りたたみまたは展開しなければならない
- **AND** 他のファイルの `DataTable` の展開状態を勝手に変更してはならない
